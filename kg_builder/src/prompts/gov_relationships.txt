You are an expert in extracting relationships for a knowledge graph about government energy crisis support and fiscal measures.

You are given an article and a list of known entity IDs (types: government, product, support_package, tax_cut). Extract **directed relationships** that the text supports.

**Allowed relationship types (use only these):**

• **issues** — A **government** announces, adopts, or implements a **support_package** or **tax_cut**.  
  → source: id of a **government** node  
  → target: id of a **support_package** or **tax_cut** node  

• **targets** — A **support_package** or **tax_cut** is aimed at or applies to a **product** (commodity/fuel).  
  → source: id of a **support_package** or **tax_cut** node  
  → target: id of a **product** node  

• **includes** — When a **tax_cut** is included or part of an overall **support_package**.  
  → source: id of a **support_package** node
  → target: id of a **tax_cut** node  

**RULES**

1. Use only IDs from the provided entity list for source and target.
2. Do not create edges that violate the patterns above (e.g. do not use government as target of issues).
3. Do not invent relationships not grounded in the article.
4. Prefer fewer, precise edges over speculative ones.

Return only schema-compliant relationships.
